Install

climate is installable through Go, GitHub Releases, and a published Homebrew tap.

Go

go install github.com/disk0Dancer/climate/cmd/climate@latest

GitHub Releases

curl -L https://github.com/disk0Dancer/climate/releases/latest/download/climate-darwin-arm64.tar.gz | tar xz
sudo mv climate-darwin-arm64 /usr/local/bin/climate

Homebrew

brew tap disk0Dancer/tap
brew install climate
Homebrew: climate lives in the published tap disk0Dancer/tap. If you want the fully qualified form, use brew install disk0Dancer/tap/climate.

Quick start

Generate a CLI from a remote OpenAPI spec, then use the generated binary as a normal command-line tool.

1. Generate a CLI

climate generate --name petstore https://petstore3.swagger.io/api/v3/openapi.json

2. Use it

petstore pet list --output=json
petstore pet get --pet-id 1 --output=json
petstore pet add --data-json '{"name":"Fido","status":"available"}' --output=json

3. Authenticate

export PETSTORE_TOKEN=my-access-token
petstore pet list --output=json

4. Inspect what climate registered locally

climate list

5. Publish the generated CLI to GitHub

export GITHUB_TOKEN=github_pat_your_token
climate publish petstore --owner disk0Dancer

Generated CLI shape

Every generated CLI follows the same mental model so people do not need to relearn the interface for each API.

<cli-name> <tag> <operation> [flags] --output=json|table|raw

Subcommands

The first level is the OpenAPI tag. The second level is the generated operation command.

Parameters

Path parameters become required flags. Query parameters become optional flags.

Request bodies

JSON bodies are passed via --data-json or --data-file.

Errors

Both CLI failures and API failures are emitted as structured JSON errors.

Agents and skills.sh

The docs site is written for humans first, but the repo also exposes machine-friendly skill assets so the climate toolchain is easy to attach to an agent.

Get the built-in climate skill

climate skill generator

Install the repo skill via skills.sh-compatible tooling

npx skills add https://github.com/disk0Dancer/climate --skill climate-generator

Generate a skill for a newly created CLI

climate skill generate petstore --mode=full
climate skill generate petstore --mode=compact

Publish a generated CLI with lifecycle bootstrap

climate publish petstore --owner disk0Dancer

The repo ships the embedded skill at skills/climate.md and a skills.sh-friendly version at skills/climate-generator/SKILL.md.

Command reference

climate generate [flags] <spec>

Generate and optionally build a CLI from an OpenAPI URL or local file.

climate list

List generated CLIs recorded in ~/.climate/manifest.json.

climate remove <name>

Remove a CLI binary and manifest entry. Use --purge-sources to delete generated source as well.

climate upgrade <name>

Regenerate and rebuild a previously generated CLI from the recorded spec or a new spec passed via --openapi.

climate publish <name>

Create or reuse a GitHub repository for a generated CLI, bootstrap lifecycle files, initialize git, and push the source tree.

climate skill generate <name>

Emit a Markdown skill prompt for a generated CLI so an agent can self-register it.

climate skill generator

Print the built-in climate skill file itself.

Distribution

For a CLI like climate, GitHub Releases, go install, and a Homebrew tap are enough. GitHub Packages are optional, not required.

Use now

Homebrew, Go install, and GitHub Releases are all live install paths today.

Release sync

Tagged releases now sync Formula/climate.rb into disk0Dancer/homebrew-tap when HOMEBREW_TAP_TOKEN is configured.

Generated CLI lifecycle

climate publish bootstraps README, CI, and release workflow files for generated CLIs before pushing them to GitHub.

Do you need GitHub Packages?

No, not for the current product shape. Add GitHub Packages only if you want bottles or OCI-style distribution there.

Current release naming

Release archives are named with hyphens, for example climate-darwin-arm64.tar.gz.

FAQ

Why does brew install climate fail?

It should work after brew tap disk0Dancer/tap. If it still fails, run brew update and then use the fully qualified name brew install disk0Dancer/tap/climate.

Should the docs site be human-first or agent-first?

Human-first. The HTML landing is documentation. Machine-friendly companions live beside it as index.md and llms.txt.

How do I enable Google Analytics?

Create a GA4 web data stream, copy the measurement ID that starts with G-, and set it in docs/site-config.js. If the ID is empty, the site makes no Google Analytics request.

Ops notes

The repository already has GitHub Pages configured in workflow mode. The site is published from the docs/ directory.

  • index.md is the Markdown companion for people and LLMs.
  • llms.txt is the short machine-readable entrypoint.
  • robots.txt keeps crawler policy explicit.
  • docs/site-config.js is where Google Analytics can be turned on safely.
  • release.yml syncs the Homebrew tap after tagged releases when HOMEBREW_TAP_TOKEN is set.
  • climate publish uses the GitHub API plus local git to create and manage generated CLI repositories.
