Open Source · Go · LLM-ready

climate — OpenAPI → production-ready CLI on Go

Generate typed, auth-aware Go CLIs from any OpenAPI 3.x spec in seconds. Built for humans and AI agents.

Quick Start GitHub

Why climate?

⚡

Single binary

One climate binary turns any OpenAPI spec into a compiled Go CLI — no templates to edit.

🔐

Auth built-in

API key (header/query/cookie), HTTP bearer & basic, OAuth2 client credentials — all wired automatically.

🤖

LLM-agent ready

JSON output everywhere. climate skill generate emits a plain-text prompt so agents can self-register any CLI as a skill.

🏗️

Strongly typed

Generated code uses Go + cobra; path/query params become typed flags with validation.

Quick Start

1. Generate a CLI from an OpenAPI spec

climate generate --name petstore https://petstore3.swagger.io/api/v3/openapi.json

2. Use the generated CLI

# List pets
petstore pet list --output=json

# Get a pet by ID
petstore pet get --pet-id 1

# Create a pet (with JSON body)
petstore pet add --data-json '{"name":"Fido","status":"available"}'

3. Authenticate

# API key via environment variable
export PETSTORE_APIKEY_API_KEY=my-secret-key
petstore pet list

# Bearer token
export PETSTORE_TOKEN=my-bearer-token
petstore pet list

4. List all generated CLIs

climate list

climate Commands

CommandDescription
climate generate <spec>Generate & build a CLI from an OpenAPI spec (URL or file).
climate listList all generated CLIs registered in the local manifest.
climate remove <name>Remove a generated CLI (add --purge-sources to also delete source).
climate upgrade <name>Re-generate a CLI from an updated spec.
climate skill generate <name>Print a Markdown skill prompt for an LLM agent to self-register the CLI.
climate skill generatorPrint the climate.generator skill file (for registering climate itself with an agent).

Flags for generate

FlagDescription
--nameOverride the generated binary name.
--out-dirDirectory to write generated Go source code.
--no-buildGenerate source only; skip go build.
--forceOverwrite an existing output directory.

Authentication

climate reads components.securitySchemes from the spec and generates auth handling for every scheme it finds. Credentials are resolved in priority order: CLI flag → ENV variable.

SchemeENV variableCLI flag
API key (header/query/cookie)<CLI>_<SCHEME>_API_KEY--<scheme>-key
HTTP Bearer<CLI>_TOKEN--token
HTTP Basic<CLI>_USERNAME / <CLI>_PASSWORD--username --password
OAuth2 client_credentials<CLI>_CLIENT_ID / <CLI>_CLIENT_SECRET--client-id --client-secret

LLM Agent Integration (skills.sh)

climate ships with a ready-to-use Markdown skill file (skills/climate.md) that lets AI agents like Claude Code register climate as a tool and generate CLIs for any API on demand.

Register climate with your agent

# Print the climate.generator skill prompt
climate skill generator

Agent workflow (automatic)

# Step 1 — agent generates a CLI
climate generate https://api.example.com/openapi.yaml

# Step 2 — agent gets a skill prompt for the new CLI
climate skill generate myapi

# Step 3 — agent follows the self-registration instructions in the prompt
# and uses the myapi CLI for all subsequent tasks

All commands output structured JSON, making them trivially parseable by any agent. Error objects always include error.code and error.message.
